# 功能实现对话示例

展示 /feat 的完整交互流程。

---

## 正常流程

```
用户: /feat

AI: 📋 当前功能清单：
    1. [ ] 用户登录
    2. [ ] 发布文章
    做哪个？

用户: 1

AI: 🎨 UI 设计: 用户登录
    布局: 垂直布局
    组件: Button, TextBox x2, HeadingTextBlock

    ┌─────────────────────────┐
    │  用户登录               │
    │  [用户名输入框]         │
    │  [密码输入框]           │
    │  [登录按钮]             │
    └─────────────────────────┘

    是否符合预期？(Y/N)

用户: Y

AI: [Build → Test → Verify → Commit]
    ✅ 用户登录 完成
    Commit: feat: 用户登录
    建议运行 /compact 清理上下文
```

## TDD 意图模式示例

```
用户: /feat → 番茄钟计时

AI: 🎨 布局设计: 番茄钟计时
    🧪 复杂度评估: 复杂逻辑 ⚠️
    建议测试: 需要测试意图

    测试意图（请确认）：
    1. 开始计时: 点击开始 → 倒计时启动
    2. 暂停功能: 点击暂停 → 倒计时停止
    3. 计时完成: 时间到0 → 触发完成事件
    4. 10秒内可撤回: 剩余<10秒 → 显示撤回按钮

    这些场景对吗？需要补充吗？

用户: 对，再加：暂停超过30分钟自动结束

AI: ✅ 测试意图已更新
    5. 暂停超时: 暂停>30分钟 → 自动结束

    📝 现在编写测试框架...
    [Test → Build → Verify → Commit]

    ✅ 番茄钟计时 完成
```
