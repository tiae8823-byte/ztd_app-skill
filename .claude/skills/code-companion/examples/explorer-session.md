# 示例：探索侧边栏并做一个小改动

展示一次完整的 /code-companion 交互。

---

## 场景：用户想了解侧边栏，并把它改宽一点

**User:** 侧边栏的代码在哪里？

**AI:**
在主视图文件中。这个文件是整个应用的主界面，侧边栏和内容区都在这里。

关键结构：
```
第 X 行：侧边栏宽度定义（如 width: 180px 或 Width="180"）
第 Y 行：内容区占剩余空间
第 Z-N 行：侧边栏内容（标题、导航按钮等）
```

> *（AI 自动追加到 PROJECT-MAP.md：侧边栏 → 主视图文件 第 Z-N 行）*

---

**User:** "占剩余空间"什么意思？

**AI:**
意思是"侧边栏占固定宽度，剩下的全给内容区"。

> *（AI 自动追加到 VOCABULARY.md：占剩余空间 = 固定宽度之外的所有空间自动分配）*

---

**User:** 我想把侧边栏变宽到 220

**AI:**
🟢 安全改动。

改主视图文件中侧边栏宽度：180 → 220。

已修改。侧边栏现在宽 220，内容区自动缩窄（因为内容区设定为占剩余空间，会重新计算）。

---

**User:** 不懂，为什么内容区会自动变？

**AI:**
想象一个 1000 宽的桌子。你放了一个 180 的盒子在左边，剩下 820 归右边。现在左边盒子变成 220，右边自动变成 780。"占剩余空间"就是"我不挑，剩多少我要多少"。

---

以上展示了：定位 → 翻译术语 → 安全评估 → 执行 → 追问加深理解 → 类比重新解释。
