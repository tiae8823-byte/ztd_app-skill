# 示例：探索侧边栏并做一个小改动

展示一次完整的 /code-companion 交互。

---

## 场景：用户想了解侧边栏，并把它改宽一点

**User:** 侧边栏的代码在哪里？

**AI:**
在 `Views/MainWindow.xaml` 第 15-50 行左右。这个文件是整个应用的主界面，侧边栏和内容区都在这里。

关键结构：
```
第 13 行：<ColumnDefinition Width="180"/>  ← 侧边栏宽度
第 14 行：<ColumnDefinition Width="*"/>    ← 内容区占剩余空间
第 16-48 行：侧边栏内容（标题、导航按钮、番茄钟）
```

> *（AI 自动追加到 PROJECT-MAP.md：侧边栏 → MainWindow.xaml 第 15-50 行）*

---

**User:** `Width="*"` 什么意思？

**AI:**
意思是"占剩余所有空间"。相当于说"侧边栏占 180 像素，剩下的全给内容区"。

> *（AI 自动追加到 VOCABULARY.md：`Width="*"` = 占剩余空间）*

---

**User:** 我想把侧边栏变宽到 220

**AI:**
🟢 安全改动。

改 `MainWindow.xaml` 第 13 行：
```xml
<!-- 改前 -->
<ColumnDefinition Width="180"/>
<!-- 改后 -->
<ColumnDefinition Width="220"/>
```

已修改。侧边栏现在宽 220 像素，内容区自动缩窄（因为 `*` 会重新算剩余空间）。

---

**User:** 不懂，为什么内容区会自动变？

**AI:**
想象一个 1000 像素宽的桌子。你放了一个 180 的盒子在左边，剩下 820 归右边。现在左边盒子变成 220，右边自动变成 780。`*` 就是"我不挑，剩多少我要多少"。

---

以上展示了：定位 → 翻译术语 → 安全评估 → 执行 → 追问加深理解 → 类比重新解释。
