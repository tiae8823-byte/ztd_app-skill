# feat 技能说明

## 这是什么？

这是 **Claude Code 的一个自定义技能**，用于**按功能开发软件**。

它把软件开发拆解成小步骤，每完成一个功能就立即测试、验证、提交，避免最后集成时出大问题。

---

## 核心流程

```
写代码 → 写测试 → 编译+测试 → 提交代码
   ↓         ↓          ↓           ↓
 修改      修改       修复        完成
```

每次只做**一个功能**，做完就 Commit。

---

## 使用场景

- ✅ 开发新功能时，确保代码质量
- ✅ 教新手养成"小步快跑"的好习惯
- ✅ 随时可以中断，下次继续（进度自动保存）
- ✅ 支持随时加新需求到 PRD

---

## 使用方法

在 Claude Code 中调用：

```
/feat
```

然后按提示操作：
1. 选择要开发的功能
2. 写代码
3. 写测试（单元 + UI 自动化 + 手动）
4. 编译运行
5. 提交代码

---

## 测试体系

这个技能有**三套测试**：

| 测试类型 | 说明 |
|---------|------|
| **单元测试** | 测试核心逻辑、边界条件、死循环防御 |
| **UI 自动化测试** | 用 FlaUI 自动测试界面交互 |
| **手动测试清单** | 14 项详细的人工测试项 |

---

## 适用项目

- ✅ WinUI 3 / WPF / WinForms 桌面应用
- ✅ .NET Core / .NET 8.0+
- ✅ MVVM 架构项目
- ✅ SQLite 数据库

---

## 实际案例

这个技能已经用于开发 **ZTD（Zen To Done）生产力应用**：

- 技术栈：WPF + .NET 8.0 + SQLite
- 已完成功能：想法收集（输入、列表、删除、数据持久化）
- 可执行文件：`ZtdApp.exe` (72MB 自包含版本)

---

## 特点

1. **进度自动保存** - 随时中断，下次继续
2. **支持恢复** - 检测到未完成功能会提示
3. **PRD 联动** - 自动更新 PRD.md 标记完成
4. **版本日志** - Git Hook 自动记录提交历史
