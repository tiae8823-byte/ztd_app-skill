# 产品需求文档 (PRD)

> 生成时间: 2026-02-27

---

## 1. 产品概述

| 项目 | 内容 |
|------|------|
| 产品名称 | Zen To Done (ZTD) |
| 产品类型 | 效率 / 时间管理 / 习惯养成 |
| 目标用户 | 追求简洁高效的个人效率用户、GTD 爱好者、极简主义者 |
| 核心价值 | 用最简单的方式，养成高效习惯，专注最重要的事 |

## 4. 用户故事

### 收集
- 作为用户，我想要快速记录想法，以免忘记
- 作为用户，我想要随时随地收集任务，即使离线也能使用

### 今日要事
- 作为用户，我想要每天只专注 1-3 件事，避免被琐事分散精力
- 作为用户，我想要看到今日要事的完成进度

### 习惯追踪
- 作为用户，我想要追踪习惯的连续天数，获得成就感
- 作为用户，我想要设置提醒，帮助我坚持习惯

### 每周回顾
- 作为用户，我想要定期回顾本周完成情况，发现改进点
- 作为用户，我想要提前规划下周的重点任务

---

## 5. 验收标准

- [ ] 用户可在 3 秒内完成一条任务收集
- [ ] 应用支持完全离线使用，联网后自动同步
- [ ] 今日要事最多显示 3 条，不可超过
- [ ] 习惯追踪支持连续天数显示和断签提醒
- [ ] 每周回顾引导式完成，包含 5 个固定问题
- [ ] 首屏加载时间 < 2 秒
- [ ] 支持 iOS 和 Android 双平台

---

## 6. 边界条件

| 场景 | 处理方式 |
|------|----------|
| 离线使用 | 数据存储本地 SQLite，联网后自动同步 |
| 同步冲突 | 以最后修改时间为准，保留本地副本 |
| 习惯断签 | 显示断签天数，不重置连续记录，鼓励继续 |
| 今日要事超额 | 提示用户保持专注，最多只能添加 3 条 |
| 数据迁移 | 支持导出 JSON/CSV 格式 |

---

## 7. 非功能需求

### 7.1 性能
- 首屏加载 < 2s
- 任务操作响应 < 100ms
- 离线可用，无网络时核心功能正常

### 7.2 安全
- 用户数据端到端加密
- Supabase JWT 认证
- 本地数据库加密存储

### 7.3 兼容性
- iOS 13.0+
- Android 8.0+
- 支持深色模式
- 支持横竖屏切换

## 8. 数据模型

### 任务 (Task)
```
- id: UUID
- title: string
- status: inbox | todo | today | done | deleted
- priority: number (1-3)
- project_id: UUID | null
- created_at: datetime
- completed_at: datetime | null
- synced: boolean
```

### 习惯 (Habit)
```
- id: UUID
- name: string
- frequency: daily | weekly
- target_days: number[] (周几)
- streak: number (连续天数)
- created_at: datetime
```

### 习惯记录 (HabitLog)
```
- id: UUID
- habit_id: UUID
- date: date
- completed: boolean
```

### 周回顾 (WeeklyReview)
```
- id: UUID
- week_start: date
- accomplishments: string[]
- challenges: string[]
- next_week_focus: string[]
- created_at: datetime
```
