## 问题记录

### 3.2.1 增量更新已有功能模块

**问题**：`/feat` 过程中，如果想为单独的功能模块增添功能，该怎么办？
例如想法收集已经实现了，但想为它做增量更新。需要在 feat 技能里说明吗？PRD 需不需要更改？

**解决方案**：feat 技能不需要修改，现有流程已支持增量更新。

**操作方式**（推荐方式 A）：

| 方式 | 操作 | 说明 |
|------|------|------|
| **A（推荐）** | `/feat 想法收集 - 增加搜索功能` | 直接带描述调用，AI 自动加到 PRD 并进入 DBTC 流程 |
| B | 先在 PRD.md 加一行 `- [ ] 想法收集 - 增加搜索功能`，再调用 `/feat` | 手动更新 PRD 后选择 |

> **注意**：直接调用 `/feat` 不带描述，只会显示 PRD 中未完成的功能列表，已完成模块的增量需求不会自动出现。

**依据**：
- feat 技能已有"过程中发现新需求"机制，支持随时往 PRD 加功能
- prd 技能明确"PRD 是活的，可以在 feat 过程中持续更新"

**结论**：feat 技能不需要修改。

### 3.2.2 PRD 技能是否全面

**问题**：prd 技能目前的功能是否全面？

**分析结果**：存在以下问题，需要修改。

| # | 问题 | 现状 | 改进方向 |
|---|------|------|---------|
| 1 | 技术栈推荐过于死板 | 只列了 4 个 Web 场景的固定推荐表 | 改为 AI 根据需求分析推荐 |
| 2 | UI 风格只覆盖 Web | 只有 shadcn/ui、Material UI、Ant Design | 根据技术栈自动匹配 |
| 3 | 缺少"调研现有方案"步骤 | 工具列表有 GitHub 搜索但流程没用上 | 增加调研环节 |
| 4 | 步骤编号重复混乱 | 步骤 3/5 重复，步骤 7 出现两次 | 重新整理流程 |

**结论**：PRD 技能需要修改。

---

### 3.2.3 技术栈推荐太死板

**问题**：指定技术栈会不会比较死板？应该让 AI 自己选择和匹配吗？

**解决方案**：是的，不应该用固定推荐表。

**改进方向**：
- 删除固定的技术栈推荐表
- 改为 AI 根据以下维度分析后推荐：
  - 产品类型（Web / 桌面 / 移动 / 跨平台）
  - 用户规模和部署方式
  - 团队技术背景和熟悉度
  - 性能和离线需求
- AI 推荐时必须说明理由，用户确认后才确定
- UI 风格同理，根据确定的技术栈自动匹配合适的方案

**结论**：需要修改 PRD 技能，删除固定表，改为分析式推荐。

### 3.2.4 应先参考成熟方案

**问题**：开发过程中应先参考网上成熟方案，避免重复造轮子。所有资料查找优先查询相关官方技术文档。

**解决方案**：PRD 和 feat 分别在不同层面加入调研。

| 阶段 | 调研什么 | 怎么做 |
|------|---------|--------|
| **PRD 阶段** | **产品设计**：同类产品的功能设计、交互模式 | WebSearch 搜索竞品、阅读产品评测 |
| **feat Build 阶段** | **技术实现**：如何编码实现当前功能 | 官方文档优先（context7）→ 网上方案（WebSearch）→ 自行实现 |

**UI 设计**：保持简单，三种风格选一个即可（简洁/Material/企业），不需要复杂调研。

**资料查找优先级**：官方技术文档 > 网上成熟方案 > 自行实现

**结论**：已修改。PRD 步骤 2 增加产品设计调研，feat Build 阶段增加技术实现调研。

---

### 3.2.5 UI 设计与技术栈的关系

**问题**：UI 设计是不是和前端绑定的？跟技术栈选择是否重复？样式设计该怎么放到开发流程里？

**分析结果**：UI 组件库/样式体系本质上是技术栈的一部分，之前的流程在三个地方做样式决策（PRD选技术栈、PRD选UI风格、feat Design选样式），存在重复。

**解决方案**：
- PRD：把"确定 UI 风格"合并到"确定技术栈"步骤（一步到位）
- feat Design：只做布局（什么组件放哪里），样式直接用 PRD 定好的方案
- feat Build：编码时直接套用已确定的样式，不再做选择

**最终流程**：
```
PRD: 需求 → 产品调研 → 技术栈+UI方案(一步) → 架构 → 功能清单
feat: 选功能 → 布局设计(Y/N) → 技术调研+编码 → 测试 → 验证 → 提交
```

**结论**：已修改 PRD 和 feat 技能。经验总结见 `docs/vibe-coding-experience.md`。
